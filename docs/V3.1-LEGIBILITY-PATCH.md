# Uoink — v3.1 legibility patch · what changed

Applied to the prototype on **(date in `CHANGELOG.md`)**, after the designer review flagged three issues with the v3 ship. The Codex prompts in this folder already reflect the new tokens, but this file is the changelog if you need to explain the diff to your agents.

## Seven changes

1. **Two-tone orange — rust primary, vermillion accent.**
   - New token: `--rust: #C2410C` is the brand colour on type, surfaces, and most headlines.
   - `--vermillion: #FF3D00` survives as the 5% accent — CTAs, hover, one stamp per viewport, hero italic emphasis.
   - **Why:** vermillion-everywhere failed AA contrast on cream (3.4 : 1). Rust passes (4.7 : 1) and stops the page from buzzing.

2. **Vermillion-trimmed ink panels (replaced `.big-strip`).**
   - The pattern that used to be a full vermillion ground is now ink with thin 3 px vermillion rules top + bottom.
   - **Why:** five vermillion-flooded sections per scroll fatigued the eye. The thin-rule version retains poster gravity at ~10% of the surface area.

3. **Magnet-U tips are size-aware.**
   - `uoink-mark.js` reads its rendered width via `ResizeObserver`. At ≤32 px, tips switch to cream (`#FFF4EC`) at 20% glyph height. Above 32 px, tips stay acid (`#FFD23F`) at 14%.
   - **Why:** acid-on-dark-Chrome at 16 px favicon scale was 2 yellow pixels — invisible. Cream tips at thicker height read clearly at every size.

4. **Type contrast house rules.**
   - Vermillion only on dark grounds.
   - Rust only on cream / ink (never on vermillion).
   - Acid only as solid shapes (stamps, magnet tips). **Never as type.**
   - **Why:** every contrast failure traced to one of these three pairings. Codified as rules so future surfaces don't re-introduce them.

5. **Three typefaces, not five. Bungee · Inter · JetBrains Mono.**
   - Dropped Fraunces and Instrument Serif. Inter 800 + Inter italic handle every display and editorial role they used to.
   - **Why:** five families = 9 font files before paint, two italics doing the same job. The brand reads cleaner with fewer voices.

6. **Discord lean — rounded, blocky, conversational.**
   - All cards: 12 px radius. Buttons: pill (`--r-pill`). Corpus panels: 8 px. Stamps stay circular.
   - New patterns: `.block-tile` (color-block sections), `.bubble` (chat-style asides), `.chat-bubble.user` and `.chat-bubble.assistant` for the agent demo.
   - **Why:** the audience is creators, not enterprise buyers. Rounded geometry + speech-bubble copy beats a Linear clone for "this is for me" recognition.

7. **Headline: "Uoink that shit."**
   - New hero default on the homepage. Subhead handles the explanation in a calmer voice.
   - The CWS-safe fallback ("Uoink any video. Read it like a doc.") + two other variants ("Uoink it. Read it." / "Yo. Uoink that.") live in the Tweaks panel for in-browser A/B.
   - **Why:** the verb is the brand's strongest asset. The previous default was correct but polite — it didn't grab.

## Telling your Codex / agents

The Codex handoff prompts (`docs/CODEX-HANDOFF.md`, `docs/RENAME-SCOPE.md`, `docs/BUTTON-DESIGN.md`) already speak v3.1. The brand v3 doc (`docs/BRAND-SYSTEM-v3.html`) still describes the original vermillion-primary system — **treat this v3.1 patch as the override** until the brand doc is reflowed (low priority; the prompts are the source of truth for the build).

Inside any agent prompt for production work, paste this block:

> Brand v3.1 patch is in effect: rust `#C2410C` is the primary surface and type colour. Vermillion `#FF3D00` is the 5% accent — used only on CTAs, hover, hero italic emphasis, and stamps. Acid `#FFD23F` is for tips and stamp interiors, never type. Type stack: Bungee (wordmark only) + Inter 400/500/600/700/800 + Inter italic + JetBrains Mono. No serifs. Default headline: "Uoink that shit." with CWS-safe fallback "Uoink any video. Read it like a doc." All radii are 8–14 px (cards 12, buttons pill, corpus 8). Discord-lean: rounded, blocky, conversational. The full diff is in `docs/V3.1-LEGIBILITY-PATCH.md` if needed.
